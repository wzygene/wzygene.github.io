Vue+axios实现登陆拦截

Nov 01, 2017点击

项目中需要实现一个登陆请求的判断，实现思路是使用 vue-router 和 axios 配合来实现这个功能。

登陆拦截的逻辑

第一步：路由拦截

定义完路由后，我们主要是通过使用vue-router提供的钩子函数beforeEach()来对路由进行判断。代码如下：

每个钩子函数接收三个参数：

  • to：Route，即将要进入的目标路由对象
  • from：Route，跳转之间的路由对象
  • next：Function，一定要调用next 方法来resolve（参见Promise）这个钩子。执行效果依赖 next 方法的调用参数：
    • next()：进行管道中的一个钩子。如果全部钩子执行完了，则导航的状态就是 comfirmed（确认的）
    • next(false)：中断当前的导航，如果浏览器的 URL 地址改变了（可能是用户手动或这后退按钮改变的），那么URL地址会重置到from参数路由对应的地址
    • next(‘/‘) 或者 next({path: ‘/‘})：调转到一个不同的地址。当前的导航被中断，然后开始一个新的导航。

确保要调用 next() 函数，否则钩子函数不会被resolved

这里的to.meta字段是我们自定义的，里面有我们用来判断是否需要登陆权限的信息。需要登录权限而当前我们没有登陆所需的 token 的情况下，会跳转到login页面，然后在登陆成功之后调转到目标路由。

拦截请求这个时候还灭有完成，现在只是做了最简单的前端路由控制，并不能真正阻止用户访问需要登陆权限的路由。还有一种情况是：当前token失效了，但是token依然保存在本地，这时候你去访问需要登陆权限的路由时，实际上应该让用户重新登陆。这就需要使用 http 拦截器和后端返回的 http 状态码来判断。

第二步：拦截器

想要统一处理所有的 http 请求和响应，就得用上 axios 的拦截器。通过配置 http resource inteceptor，当后端接口返回 401 Unauthorized（未授权），让用户重新登陆。

通过上面的这两步，就可以在前端实现登陆拦截了。登出功能也就很简单，只需要把当前token清除，然后再跳转到首页即可。
