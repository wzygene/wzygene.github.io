Flex布局（伸缩式布局）

Oct 18, 2016点击

1. 前言

页面开发是Web前端开发的基础，虽说是基础，但也不可小视。

布局的传统解决方案，基于盒状模型，依赖 display 属性 + position属性 + float属性。它对于那些特殊布局非常不方便，比如，垂直居中就不容易实现。

直到2009年，W3C 提出了一种新的方案—-Flex 布局，即伸缩布局盒模型（Flexible Box）,用来提供一个更加有效的方式制定、调整和分布一个容器里内部元素布局，即使它们的大小是未知或者动态的，简称Flex。

2. 特点

Flexbox布局常用于比较复杂的页面，可以保证屏幕和浏览器窗口大小发生变化时保持元素的相对大小和位置不变，同时减少了依赖于浮动布局实现元素位置的定义以及重置元素的大小。

Flex布局在定义伸缩内部元素大小时伸缩容器会预留一些可用空间，让你可以调节伸缩内部元素的相对大小和位置。例如，你可以确保伸缩容器的多余空间平均分配给多个伸缩内部元素，当然，如果你的伸缩容器没有足够大的空间放置伸缩内部元素时，浏览器会根据一定的比例减少伸缩内部元素的大小，使其不溢出伸缩容器。综合而言，Flexbox布局功能主要具有以下几点：

  1. 屏幕和浏览器窗口大小发生改变时可以灵活调整布局；
  2. 可以指定元素沿着主轴或侧轴按比例分配额外空间（伸缩容器额外空间），从而调整伸缩内部元素的大小；
  3. 可以指定元素沿着主轴或者侧轴将伸缩容器额外空间，分配到元素之前、之后或之间；
  4. 可以指定如何将垂直于元素布局轴的额外空间分布到该元素的周围；
  5. 可以控制元素在页面上的布局方向；
  6. 可以按照不同于文档对象模型（DOM）所指定排序方式对屏幕上的元素重新排序。也就是说可以在浏览器渲染中不按照文档流先后顺序重排伸缩内部元素顺序。

3. 概念

采用 Flex 布局的元素，称为 Flex 容器（flex container），简称”容器”。它的所有子元素自动成为容器成员。

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）。主轴的开始位置（与边框的交叉点）叫做main start，结束位置叫做main end；交叉轴的开始位置叫做cross start，结束位置叫做cross end。

内部元素默认沿主轴排列。单个元素占据的主轴空间叫做main size，占据的交叉轴空间叫做cross size。

4. 属性

以下6个属性设置在Flex容器上：

  • flex-direction
  • flex-wrap
  • flex-flow
  • justify-content
  • align-items
  • align-content

4.1 flex-direction属性

flex-direction属性决定主轴的方向（即内部元素的排列方向）

4.2 flex-wrap属性

默认情况下，内部元素都排在一条线（又称”轴线”）上。flex-wrap属性定义，如果一条轴线排不下，如何换行

他有三个属性值：

  1. nowrap（默认）：不换行
  2. wrap：换行，第一行在上方
  3. wrap-reverse：换行，第一行在下方

4.3 flex-flow属性

flex-flow属性是flex-direction属性和flex-wrap属性的简写形式，默认值为row nowrap

4.4 justify-content属性

justify-content属性定义了内部元素在主轴上的对齐方式

它可能取5个值，具体对齐方式与轴的方向有关。下面假设主轴为从左到右

  • flex-start（默认值）：左对齐
  • flex-end：右对齐
  • center： 居中
  • space-between：两端对齐，内部元素之间的间隔都相等。
  • space-around：每个内部元素两侧的间隔相等。所以，内部元素之间的间隔比内部元素与边框的间隔大一倍。

4.5 align-items属性

align-items属性定义内部元素在交叉轴上如何对齐

它可能取5个值。具体的对齐方式与交叉轴的方向有关，下面假设交叉轴从上到下。

  • flex-start：交叉轴的起点对齐。
  • flex-end：交叉轴的终点对齐。
  • center：交叉轴的中点对齐。
  • baseline: 内部元素的第一行文字的基线对齐。
  • stretch（默认值）：如果内部元素未设置高度或设为auto，将占满整个容器的高度。

4.6 align-content属性

align-content属性定义了多根轴线的对齐方式。如果内部元素只有一根轴线，该属性不起作用

该属性可能取6个值。

  • flex-start：与交叉轴的起点对齐。
  • flex-end：与交叉轴的终点对齐。
  • center：与交叉轴的中点对齐。
  • stretch（默认值）：轴线占满整个交叉轴。
  • space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。
  • space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。

5. 内部元素的属性

以下6个属性设置在内部元素上。

  • order
  • flex-grow
  • flex-shrink
  • flex-basis
  • flex
  • align-self

5.1 order属性

order属性定义内部元素的排列顺序。数值越小，排列越靠前，默认为0

5.2 flex-grow属性

flex-grow属性定义内部元素的放大比例，默认为0，即如果存在剩余空间，也不放大

5.3 flex-shrink属性

flex-shrink属性定义了内部元素的缩小比例，默认为1，即如果空间不足，该内部元素将缩小

5.4 flex-basis属性

flex-basis属性定义了在分配多余空间之前，内部元素占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即内部元素的本来大小

它可以设为跟width或height属性一样的值（比如350px），则内部元素将占据固定空间

5.5 flex属性

flex属性是flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto。后两个属性可选

该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。

建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值

5.6 align-self属性

align-self属性允许单个内部元素有与其他内部元素不一样的对齐方式，可覆盖align-items属性。默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch
